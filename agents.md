# AI Guidance Rules

## Purpose
This file defines how AI tools were used during the development of this project.

---

## AI Usage Approach
- AI was used for generating boilerplate code and debugging issues
- All AI-generated code was manually reviewed before use
- No code was used without understanding its functionality

---

## Constraints
- Avoid overly complex logic
- Keep code simple and readable
- Follow REST API design principles
- Ensure proper error handling

---

## Validation Rules
- Ensure all API inputs are validated
- Prevent invalid states (e.g., empty tasks)
- Handle missing or incorrect data gracefully

---

## Coding Standards
- Use clear and meaningful variable names
- Keep functions small and focused
- Maintain separation between frontend and backend

---

## Safety Measures
- Do not expose sensitive data
- Avoid using unverified external code
- Ensure predictable and stable behavior

---

## AI Responsibility Statement
AI was used as an assistance tool, not as a replacement for understanding. All code was tested and verified manually.
